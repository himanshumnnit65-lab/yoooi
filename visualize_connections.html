Concurrency 2. Redis Pub/Sub Parallel Execution Sequence

This sequence diagram highlights how an incoming request is dispatched simultaneously to 5 independent worker containers via Upstash Redis Pub/Sub channels, achieving 3-5x faster responses than sequential execution, and how streaming progress updates push directly to the user over WebSockets.

AI Ecosystem 3. Model Context Protocol (MCP) Dynamic Tool Binding

To prevent LLM context window bloat and tool hallucination when exposing 38 custom travel tools, TBuddy agents query the FastMCP server (`mcp_server.py`) at runtime via `get_mcp_tools`, inspect schemas by category tag, and dynamically bind only the tools required for their specific task.

Interactive UX 4. Human-in-the-Loop (HITL) State Machine & Delta Loop

When a user interacts with the 5 high-impact features (Thumbs Up/Down, Budget Sliders, Activity/Day Locks, Preference Polls, or Swap Suggestions), the system mutates the `HITLState` in Redis and performs targeted delta-regeneration without recomputing unlocked portions of the trip.

Algorithmic Math 5. Hybrid Neuro-Symbolic Routing (LLM + OR-Tools TSP)

LLMs are creative reasoning engines but fail at mathematical combinatorial optimization. TBuddy uses a Hybrid Architecture: the LLM selects theme-aligned activities, while Google OR-Tools solves the Traveling Salesperson Problem (TSP) to order locations geographically, minimizing travel time and distance.
